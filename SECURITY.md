# Security Policy

This repository is documentation-first, but security still matters.

## Please do not include

- API keys, tokens, passwords, cookies, private prompts, or customer data.
- Exploit instructions for systems you do not own or have permission to test.
- Instructions that encourage agents to bypass safety gates around secrets, payments, production, or destructive commands.

## Reporting

Open a private security advisory if the repository enables accidental secret exposure or unsafe automation. If advisories are unavailable, open a minimal public issue without sensitive details.
