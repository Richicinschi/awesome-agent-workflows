# Curation Policy

A resource or workflow belongs here when it is useful to someone operating AI agents on real work.

## Include

- Practical workflows with clear trigger, steps, and verification.
- Tooling that helps agents plan, execute, review, test, or recover.
- Examples that show real-world agent operations, not only toy demos.
- Safety patterns for secrets, production, public release, and destructive actions.

## Exclude

- Pure marketing pages.
- Unverified claims without a usable artifact.
- Prompt dumps without context or guardrails.
- Spam, referral links, or paid placement.

## Accepted examples

- A workflow that shows how to split a feature across three agents and integrate the result.
- A checklist for reviewing agent-generated code before merge.
- A tool adapter that explains exactly where to paste the brief and how to verify output.
- A postmortem of an agent failure with the guardrail that would have prevented it.

## Rejected examples

- A list of prompts with no trigger, scope, verification, or failure modes.
- A vendor marketing page with no operational guidance.
- A resource that asks users to paste secrets into an agent prompt.
- A growth tactic based on fake stars, star exchanges, mass comments, or spam.

## Field-tested language

Use “field-tested” only when a workflow has been used against a real project, incident, review, or repo operation. If the workflow is proposed but not yet exercised, label it as a candidate or draft.

## Preferred contribution style

Add a short explanation of **why** the resource is useful. A raw link list is less valuable than an operational note.
